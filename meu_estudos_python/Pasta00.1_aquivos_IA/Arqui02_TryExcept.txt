O Que é ValueError?
O ValueError é uma exceção (um tipo de erro) que o Python levanta quando uma função ou operação recebe um argumento que tem o tipo correto, mas um valor inapropriado.

Pense assim:

Você esperava um número inteiro, mas o usuário digitou "abc".
int("abc") -> Isso levanta um ValueError, porque "abc" não pode ser convertido para um inteiro, mesmo que a função int() espere uma string para tentar a conversão. 
O tipo (string) estava certo, mas o valor não.
Você esperava um número em um determinado intervalo (por exemplo, de 1 a 10), e o usuário digitou 15.
Se seu código tiver uma validação if numero > 10: raise ValueError("Número fora do intervalo"), ele levantaria um ValueError.
No seu código:

Quando o usuário digita algo como "teste" no input() e você tenta fazer int("teste"), o Python não consegue converter "teste" para um número inteiro. 
O int() está esperando uma string que represente um número. 
Como "teste" não representa, ele levanta um ValueError.

Existem Outros Tipos de Erro? Sim, Vários!
O Python tem uma hierarquia rica de exceções (erros) para diferentes situações. O ValueError é apenas um deles. 

Veja alguns exemplos comuns:

TypeError: Ocorre quando uma operação ou função é aplicada a um objeto de tipo inapropriado.

Exemplo: len(5) (você tenta obter o comprimento de um número, que não tem comprimento).
Exemplo: "hello" + 5 (você tenta somar uma string com um número).
NameError: Ocorre quando você tenta usar um nome (variável, função) que não foi definido.

Exemplo: print(minha_variavel) antes de minha_variavel = 10.
IndexError: Ocorre quando você tenta acessar um índice fora dos limites de uma sequência (como uma lista ou tupla).

Exemplo: minha_lista = [1, 2, 3]; print(minha_lista[3]) (a lista só vai até o índice 2).
KeyError: Ocorre quando você tenta acessar uma chave que não existe em um dicionário.

Exemplo: meu_dicionario = {"nome": "Maria"}; print(meu_dicionario["idade"]).
FileNotFoundError: Ocorre quando você tenta abrir um arquivo que não existe.

Exemplo: with open("arquivo_inexistente.txt", "r") as f: pass.
ZeroDivisionError: Ocorre quando você tenta dividir um número por zero.

Exemplo: 10 / 0.
Como try-except Funciona com Diferentes Erros?
O bloco try-except é a ferramenta do Python para lidar com esses erros de forma elegante, evitando que seu programa "quebre".

Python

try:
    # Código que pode levantar um erro (exceção)
    numero = int(input("Digite um número: "))
    resultado = 10 / numero
    minha_lista = [1, 2]
    print(minha_lista[numero])
except ValueError:
    # Este bloco é executado SE um ValueError ocorrer no bloco try
    print("Isso não é um número válido!")
except ZeroDivisionError:
    # Este bloco é executado SE um ZeroDivisionError ocorrer
    print("Não consigo dividir por zero!")
except IndexError:
    # Este bloco é executado SE um IndexError ocorrer
    print("Você tentou acessar um índice inválido na lista!")
except Exception as e:
    # Este é um "catch-all" (pega tudo). Ele captura qualquer outro tipo de erro
    # 'as e' permite que você veja a mensagem de erro específica.
    print(f"Ocorreu um erro inesperado: {e}")
Pontos Chave:

Você pode ter múltiplos blocos except para lidar com erros específicos de forma diferente. 
Isso é uma boa prática, pois permite dar mensagens de erro mais precisas ao usuário.
O except Exception as e: é um "coringa" que pega qualquer erro que não tenha sido pego por um except mais específico. 
Use-o com moderação, pois ele pode mascarar problemas que você deveria tratar especificamente. 
É útil para depuração ou para garantir que o programa não crash.
A ordem dos except importa: os mais específicos devem vir antes dos mais genéricos. 
Por exemplo, except ValueError antes de except Exception.
Para sua dúvida específica sobre str e booleanos:

ValueError para strings: Como você viu, acontece quando uma string não pode ser convertida para o tipo esperado (ex: "olá" para int).
ValueError para booleanos: Raramente ocorre diretamente em conversões simples, 
pois bool() é bastante flexível (bool(0) é False, bool(1) é True, bool("") é False, bool("texto") é True). 
Mas você pode criar um ValueError se estiver esperando uma string específica para representar um booleano 
(ex: "True" ou "False") e o usuário digitar outra coisa.
Em resumo, try-except é seu amigo para criar programas que são mais amigáveis e resilientes a erros do usuário ou a condições inesperadas no ambiente.
